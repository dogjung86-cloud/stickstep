# 중2 V 「식물과 에너지」(g2u5) 랩 재제작 브리프 — codex 자율 작업 지시서

너(codex)는 **이미 존재하는** 대단원 중2 V 식물과 에너지(id `g2u5`)의 **랩(핵심 상호작용)만 재설계**한다.
콘텐츠 텍스트·훅·recap·문제·인프라는 이미 좋다 — **건드리지 마라.** 손댈 것은 **랩 렌더러 5개 파일뿐**이다.
이 문서 + `CLAUDE.md`(헌법) + `LESSON_GUIDE.md` + `.claude/skills/build-unit/SKILL.md`(플레이북)만 읽으면
규칙이 전부 있다. **충돌하면 CLAUDE.md가 이긴다.** 교과서 원문 범위는 책 168~197쪽(V 단원 전체).

## 0. 왜 재작업하는가 (진단 — 이걸 이해 못 하면 또 똑같이 만든다)

현재 g2u5의 랩 6종 중 **5종이 "버튼을 순서대로 누르면 애니메이션이 재생되는" 감상형**이다.
학생은 손으로 개념을 **조작**하는 게 아니라 정해진 순서대로 버튼을 누르고 **구경**한다. 실측(qa/e2e-g2u5.mjs):

| 레슨 | 랩 | 현재 조작 | 판정 |
|---|---|---|---|
| L1 | leafFactoryLab | 물→이산화 탄소→빛→광합성 시작→녹말 저장 (버튼 5연타) | ✗ 감상형 |
| L2 | photoEvidenceLab | 센서 관찰→암처리→탈색→헹굼→아이오딘 (버튼 5연타) | ✗ 감상형 |
| **L3** | **photoFactorLab** | **슬라이더로 빛·CO₂·온도를 하나씩 바꿔 곡선 관찰** | **✓ 유일하게 제대로 됨** |
| L4 | plantRespireLab | 포도당 넣기→산소 넣기→호흡 시작 (버튼 3연타) | ✗ 감상형 |
| L5 | dayNightLab | "강한 낮" / "빛 없는 밤" 사진 2장 토글 | ✗✗ 가장 얇음 |
| L6 | sugarJourneyLab | 잎에서 포도당→밤 설탕→어린잎→열매→뿌리 (버튼 5연타) | ✗ 감상형 |

플레이북 §0.1의 기준 질문 — **"이 개념이 눈에 보이려면 학생이 무엇을 직접 움직여야 하는가?"** — 을
5종 모두 통과 못 한다. 버튼 순서 클릭은 조작이 아니다. 기준(상태변화=메타볼, 위상=3D 명암,
대륙이동=시대 스크럽, 부력=수조 들어올리기)처럼 **연속 변수를 손으로 움직여 개념이 실시간으로 반응**해야 한다.

**그래서 이번 작업 = L1·L2·L4·L5·L6 랩 5종을 "조작형"으로 재설계한다. L3는 잘 됐으니 유지(§2.3).**

## 1. 절대 원칙 (위반 = 실격)

1. **손대는 파일은 랩 렌더러 5개뿐**: `src/lessons/steps/{plantFactoryLab, plantEvidenceLab,
   plantRespirationLab, sugarJourneyLab}.ts` + `plantRespirationLab.ts` 안의 `dayNightLab`.
   추가로 **plantKit.ts·plantFigures.ts에 헬퍼/그림을 append**하는 건 허용(기존 것 삭제·시그니처 변경 금지).
   **content/g2/unit5.ts·dsl.ts·registry.ts·curriculum.ts·home.ts·tokens.css·ui.css는 건드리지 마라.**
   (dsl 시그니처가 이미 `{title, lead?, cta?, curio?}`라 콘텐츠 호출부는 그대로 컴파일된다 — 이게 핵심.
   랩 lead 문구가 새 조작과 안 맞으면 **그 lead 문자열 한 줄만** unit5.ts에서 고쳐도 된다. 그 외 unit5 수정 금지.)
2. **랩 하나 완성 → `npx tsc --noEmit` 통과 → 다음.** 5개를 반쯤 만들기 금지. 순서: L5(가장 얇음, 효과 큼) → L1 → L4 → L2 → L6.
3. **기준 파일을 열어 공학 문법을 그대로 베낀다**(§3). 재설계 금지 대상은 **뼈대**(랩 레이아웃·cleanup·
   채점 플로우)고, 재설계 대상은 **상호작용 아이디어**뿐. buoyancyLab.ts가 캔버스 슬라이더 랩의 원전이다.
4. **완전 한글 해요체**, 이모지 금지(픽토그램은 SVG만), UI 문구에 '교과서'·출판사명 금지.
5. **손코딩 SVG·캔버스 소품은 파운드리 재질 문법**: 3스톱 그라데이션 면 + 좌상단 키라이트 + 접촉 그림자
   + 재질별 최암색 외곽선(1.4~1.6px). 균일 검은 외곽선 = 실격. 캔버스 소품은 `ui/labProps.ts`·`plantKit.ts`만 쓴다.
6. **이미지는 SVG 전용이 아니다 — 필요하면 발주한다**(§4). 실사 식물·온실·저장 식품 같은 건 발주가 옳고,
   기공·엽록체·그래프·화살표처럼 **판정이 필요한 구조**만 벡터로 그린다. 대부분의 발주 에셋은 **이미 존재**하니
   재사용이 1순위. 새 조작에 없는 베이스가 필요할 때만 발주(§4의 파이프라인·사고 #16 수정 포함).

## 2. 레슨별 랩 재설계 명세 (이 단원의 창작 핵심 — 정독)

각 랩 공통 계약(buoyancyLab이 기준):
- 레이아웃 순서 **`host.append(h1 → sub → pn-badges 목표칩(3개) → helper → stage)`** — helper(지시)는 조작 요소 위.
- 목표 3개를 `collect()`로 모으면 `api.recordQuiz(true)` + `api.enableCTA(step.cta ?? "…")`. 마운트 직후 `api.setCTA(…, {enabled:false})`.
- 캔버스면 `createLoop` + 매 프레임 `fitCanvas(canvas, H)`(DPR 1.75). **포인터 캡처는 plantKit의 `safePointerCapture()`만 쓴다**(생 setPointerCapture 금지 — 합성 이벤트에서 throw하면 드래그 전체가 죽는다).
- cleanup에서 loop.stop·리스너·타이머 전부 해제. curio는 랩 맨 아래에 붙는다(dsl이 처리 — 네가 신경 쓸 것 없음, `s.curio` 그대로 둠).
- 조작은 **예측→실행→확인**을 우선. 목표는 "버튼 순서 밟기"가 아니라 "연속 변수를 움직여 상태가 변하는 걸 관찰"이어야 한다.

### 2.1 L5 dayNightLab — 「24시간 기체 수지 다이얼」 (최우선·간판)
- **개념(책 184~185쪽)**: 낮=광합성>호흡(순 CO₂ 흡수·O₂ 방출), 밤=호흡만(순 CO₂ 방출), 그리고 결정적으로
  **순 기체 이동은 두 과정의 합(차)**이다. 흐린 낮처럼 광합성량이 작으면 순변화가 밤과 비슷해질 수 있고,
  **광합성량 = 호흡량이 되어 순 기체 이동이 0이 되는 빛의 세기**가 존재한다.
- **왜 이 조작인가**: 현재는 사진 2장 토글이라 "낮=O₂, 밤=CO₂"만 암기시킨다. 순변화가 **연속 변수(빛의 세기)의
  함수**임을 감추면 이 단원의 핵심을 놓친다. 학생이 다이얼을 돌려 **화살표 방향이 뒤집히는 경계**를 직접 찾아야 한다.
- **무대·조작(캔버스, buoyancyLab 슬라이더+실시간 그래프 문법 그대로)**:
  - 세로 슬라이더/다이얼 하나 = **빛의 세기**(0=한밤 → 최대=쨍한 정오). 드래그로 연속 조절(러버밴딩 권장 — core/rubber.ts).
  - 화면에 두 개의 **속도 막대**: 광합성 rate(∝ 빛의 세기, 위로 볼록하게 포화) + 호흡 rate(빛과 무관한 **상수**).
  - 식물 둘레의 **순 기체 화살표**: 광합성>호흡이면 CO₂ 흡입·O₂ 방출(초록/하늘 방향), 광합성<호흡이면 그 반대(회보라).
    화살표 굵기 = |광합성 − 호흡|. 두 값이 같아지면 화살표가 **사라진다**(순변화 0).
  - 하단 실시간 그래프(선택): 빛의 세기(x) 대 순 기체 이동(y), 0을 가로지르는 곡선 — 방문한 지점이 점으로 찍힘(buoyancyLab samples 문법).
- **목표 칩 3개**: ① 쨍한 낮 만들기(순 O₂ 방출 관찰) ② 한밤 만들기(순 CO₂ 방출) ③ **순 기체 이동이 0인 빛의 세기 찾기**(경계).
- **언어 가드**: '보상점'·'보상 상태' 용어 **금지**(중2 미도입). "광합성량과 호흡량이 같아 순 기체 이동이 없는 세기"로 서술.
  '순변화/순 기체 이동'은 굵은 화살표 = 두 과정의 합이라는 뜻으로만.
- **기준 파일**: buoyancyLab.ts(슬라이더·수렴·실시간 그래프·목표 판정 f>… 구간). 배경에 낮/밤 식물이 필요하면
  기존 `plant/figs/day-observatory.webp`·`night-observatory.webp`를 캔버스 뒤에 크로스페이드(빛의 세기에 따라 alpha).

### 2.2 L1 leafFactoryLab — 「엽록체 반응로: 무엇이 진짜 재료인가」
- **개념(책 172~173쪽)**: 빛에너지로 CO₂+물 → 포도당+산소, 포도당→녹말 저장. **핵심 오개념 = 빛이 재료다 / 흙을 먹는다.**
- **왜 이 조작인가**: 현재는 물·CO₂·빛을 정해진 순서로 넣으면 무조건 성공한다 — "빛은 에너지지 재료가 아니다"를
  **체험으로 반증할 수 없다**. 학생이 재료를 **빼 보고** 반응이 멈추는 걸 봐야 개념이 손에 잡힌다.
- **무대·조작(캔버스 입자 모형 — ui/thermo·gasKit의 발광 입자 문법 참고, 엽록체는 판정 구조라 벡터로)**:
  - 두 공급원: **기공**(CO₂ 토큰 방출) + **물관**(물 토큰 방출). 각 공급 밸브를 학생이 연다/닫는다(토글 또는 드래그로 유량 조절).
  - **빛 스위치/슬라이더**(에너지). 엽록체 반응 구역으로 CO₂·물 입자가 드래그되거나 흘러들어간다.
  - 반응은 **빛 ON + CO₂ + 물이 모두 있을 때만** 진행 → 포도당 알갱이(→녹말로 뭉쳐 저장) + 산소(기공으로 방출).
  - 재료를 빼거나 빛을 끄면 **생성 정지**를 실시간으로 보여준다.
- **목표 칩 3개**: ① 물·CO₂ 공급 + 빛 켜기 → 포도당·산소 생성 ② **빛만 켜고 재료를 끊기** → 아무것도 안 생김
  (빛은 재료가 아니다) ③ 만들어진 포도당을 **녹말로 저장**(물에 안 녹는 형태로 잎에 모음).
- **기준 파일**: buoyancyLab.ts(뼈대·목표), ui/thermo·gasKit(발광 입자), plantKit(엽록체·기공·물관 색·소품).
  기존 `plant/labs/leaf-factory-*.webp` 베이스를 캔버스 뒤에 깔아도 되고, 엽록체는 벡터로 얹는다.

### 2.3 L3 photoFactorLab — 유지 (재작업 대상 아님)
- 이미 슬라이더 변인통제 + 3곡선(빛·CO₂·온도) + 제한요인으로 **제대로 된 조작형**이다. **그대로 둔다.**
- 여유가 있으면 **제한요인 시연**만 보강: "빛이 충분한데 CO₂가 부족하면 곡선이 평평"을 두 조건 동시 표시로 한 번 더 못박기.
  (선택 사항 — 시간 없으면 손대지 마라. tsc만 통과하면 됨.)

### 2.4 L4 plantRespireLab — 「마이토콘드리아 발전소: 에너지를 꺼내 쓴다」
- **개념(책 182~185쪽)**: 포도당+산소 → CO₂+물 + **에너지**. 마이토콘드리아에서. **낮·밤 무관, 모든 살아있는 기관.**
  핵심 = 호흡은 기체 교환이 전부가 아니라 **양분에서 생명활동 에너지를 꺼내는 과정**이다.
- **왜 이 조작인가**: 현재는 포도당·산소를 넣고 "호흡 시작"을 누르면 끝 — 에너지의 **쓰임**이 안 보인다.
  학생이 꺼낸 에너지를 **직접 소비**해야 "왜 호흡하나"가 손에 잡힌다.
- **무대·조작(L1의 역과정 — 자원 루프)**:
  - 포도당+산소를 마이토콘드리아에 공급 → CO₂+물 방출 + **에너지 게이지**가 찬다.
  - 학생이 게이지의 에너지를 **생명활동에 배분**: 자라는 어린싹 탭(생장), 물질 운반, 꽃·열매 만들기 — 각각 에너지를 소비.
  - **낮/밤 토글**: 빛을 꺼도(밤) 호흡은 계속됨을 보여준다(광합성만 멈춤).
- **목표 칩 3개**: ① 포도당·산소로 에너지 꺼내기(CO₂·물 생성 확인) ② 꺼낸 에너지를 생명활동에 쓰기(생장/운반)
  ③ 밤에도 호흡이 계속됨 확인(빛 꺼도 발전소 가동).
- **기준 파일**: buoyancyLab.ts(뼈대), L1 반응로와 소품 공유(plantKit). respirationCycleFig(기존)와 개념 연결.

### 2.5 L2 photoEvidenceLab — 「광합성 탐정: 증거를 직접 세팅한다」
- **개념(책 174~175쪽)**: 두 증거 — 기체 센서(CO₂↓·O₂↑) + 아이오딘 녹말 검출. **변인통제(암처리, 잎 일부 가리기).**
- **왜 이 조작인가**: 현재는 실험 순서를 버튼으로 밟기만 한다 — 왜 암처리하고 왜 일부만 가리는지 **선택으로 체험** 못 한다.
- **무대·조작(두 실험, 캔버스+DOM 혼합)**:
  - **센서 실험**: 빛 세기 슬라이더 → CO₂/O₂ 곡선이 실시간으로 갈라진다. 빛을 끄면 곡선이 평평(대조). buoyancyLab 그래프 문법.
  - **아이오딘 실험(핵심 조작)**: 학생이 **잎 가리개를 드래그**해 잎 일부를 덮고, **암처리 여부를 토글**한 뒤 검출 →
    빛 받은 부분만 청람색, 가린 부분은 그대로. **암처리를 건너뛰면 두 부분 다 반응**(교란) → 왜 암처리가 필요한지 체험.
- **목표 칩 3개**: ① 빛 세기를 바꿔 센서 곡선 갈라짐 관찰 ② 잎 가리개 놓고 검출 → 빛 부분만 녹말 ③ 암처리 없이 해 보고
  결과가 교란됨 확인(공정한 비교의 필요).
- **기준 파일**: buoyancyLab.ts(센서 그래프), starchTestFig·sensorGraphFig(기존 그림 재사용/확장). 가리개 드래그는 `safePointerCapture()`.

### 2.6 L6 sugarJourneyLab — 「양분 배송망: 위로도 아래로도」
- **개념(책 186~197쪽)**: 포도당→녹말(낮 임시저장)→설탕(밤 이동형태)→**체관 이동(양방향)**→기관별 저장·이용
  (열매=당, 뿌리=녹말, 씨=단백질/지방; 호흡·생장에 이용). **핵심 오개념 = 체관은 아래로만 이동한다.**
- **왜 이 조작인가**: 현재는 잎→열매→뿌리로 정해진 순서 버튼. 학생이 **직접 배송지를 골라 위·아래로 보내는** 조작이
  없으면 "양방향 이동"이 안 박힌다. 훅(열매 솎기)의 보상도 여기서 회수해야 한다.
- **무대·조작(식물 전체 뷰 — 발주 베이스 `plant/figs/whole-plant.webp` 재사용 + SVG 체관 오버레이)**:
  - **낮/밤 전환**: 낮엔 잎에 녹말이 쌓이고, 밤엔 설탕으로 바뀌어 체관에 실린다(토글/탭).
  - **배송(핵심 조작)**: 설탕 화물을 체관을 따라 **학생이 목적지로 라우팅** — 어린잎·꽃(위쪽), 열매, 뿌리(아래쪽).
    각 기관 도착 시 저장/이용 형태 표시(열매=당, 뿌리=녹말, 씨=단백질·지방). **위로도 보낼 수 있어야 한다.**
  - (선택) 열매 솎기 보상: 열매 수를 줄이면 남은 열매로 배분량↑.
- **목표 칩 3개**: ① 낮에 녹말로 임시저장 → 밤에 설탕으로 전환 ② 위쪽 기관(어린잎/꽃)으로 배송 ③ 아래쪽 기관(뿌리/열매)으로 배송
  (양방향 확인).
- **기준 파일**: driftLab.ts(오버레이·경로), whole-plant.webp(기존 베이스), plantKit(체관·물관·양분 토큰 색). 라우팅 드래그는 `safePointerCapture()`.

## 4. 이미지 — SVG 전용이 아니다. 필요하면 발주한다 (사용자 지시)

**원칙(플레이북 §5)**: 판정이 필요한 구조(기공·엽록체·그래프·화살표·순 기체 방향)는 **벡터**로,
실사 질감이 교육적으로 나은 것(식물 전체·온실·저장 식품·낮밤 관찰실·스틱맨 컷)은 **발주**로.

**대부분 이미 있다 — 재사용이 1순위**(기록: `qa/g2u5-image-prompts.md`):
- `plant/figs/whole-plant.webp`(L6 배송 베이스), `greenhouse.webp`(L3), `day-observatory.webp`·`night-observatory.webp`(L5 배경),
  `storage-foods.webp`(L6 문제), `plant/labs/leaf-factory-*.webp`(L1 베이스), `plant/cuts/g2u5l1~l6.webp`(개념 컷).

**발주가 필요하면**(새 조작에 없는 베이스가 생길 때만):
1. **먼저 이 인프라 결함부터 고쳐라(사고 #16 — 이게 안 고쳐져 있다)**: `qa/process-geo.mjs`의 `ASPECT_DIRS`에
   **`public/plant/figs`·`public/plant/cuts`·`public/plant/labs`가 빠져 있다.** 세 경로를 배열에 추가하라.
   안 그러면 새로 발주한 png가 webp로 변환되지 않아 "발주했는데 이미지가 없다"가 된다.
2. `qa/g2u5_prompts.txt` 작성 — 맨 위 공통 스타일 블록 + `[i] file=public/plant/…` 항목별 영문 프롬프트.
   **이미지 안 글자·숫자·화살표·라벨 절대 금지**를 공통·개별 양쪽에 명시. 종횡비 명시.
3. `qa/order-plant.sh` 작성 — `qa/order-cuts.sh`를 복제해 만든다. 핵심 형식:
   ```bash
   codex exec --skip-git-repo-check -s danger-full-access -C "$(pwd)" - <<'PROMPT'
   qa/g2u5_prompts.txt 를 읽어라. [i] 프롬프트에 공통 스타일 블록을 붙여 내장 image_gen으로 생성하라.
   Google 도구 금지, 내장 image_gen만. 저장 경로는 각 file= 그대로. 이미지 안 글자·숫자 절대 금지.
   각 이미지 후 "IMG i: SAVED <경로>", 끝나면 "DONE n/N" 출력.
   PROMPT
   ```
   `-s workspace-write`는 네트워크가 막히니 **`danger-full-access` 필수.** **발주는 순차만**(병렬 codex는 tmp/ 충돌 — 사고 #13).
4. 백그라운드 실행 → 완료 후 **Read 도구로 눈 검수**(글자 없음·과학 정확) → `node qa/process-geo.mjs`로 webp 변환(원본 png 삭제) → 콘텐츠에 임베드.
5. 발주 못 만드는 것: 드래그 스냅용 공유 경계 벡터, 판정 구조(엽록체·기공) — 이건 SVG로.

## 5. 검증 게이트 (각 단계 끝 필수)

- 랩 하나 완성마다 `npx tsc --noEmit` 통과. 5종 다 끝나면 `npm run build`도 통과.
- **e2e는 이미 있다: `qa/e2e-g2u5.mjs`.** 네가 랩 조작을 바꾸면 이 파일의 해당 헬퍼(dayNight·leafFactory·
  photoEvidence·plantRespire·sugarJourney)도 **새 조작에 맞게 갱신**해야 한다(다음 실행자가 쓴다). L3 photoFactor 헬퍼는 그대로.
  - **프리뷰 실플레이는 다른 세션 dev 서버와 충돌하니 하지 말고**, tsc+build 통과 + 코드 자기검토로 갈음한다(메인이 e2e를 돌린다).
  - 단, 각 랩의 **목표 칩 3개가 조작으로 실제 켜지고 CTA가 열리는 경로**를 코드로 자기검토(collect 3회 → recordQuiz+enableCTA)하라.
- 언어 grep: 바뀐 파일에 '교과서' 0건, 이모지 0건, L5에 '보상점' 0건.
- 발주했다면: png 삭제·webp만 존재, 이미지 안 글자 없음, process-geo에 폴더 등록 확인, `loading="lazy"` 0건.
