이 문서가 유일한 정보원이다. **다른 파일(CLAUDE.md·MATH_GUIDE.md·교과서 PDF·레슨 파일 unit6.ts·
다른 시험 풀 파일·mathFigures2.ts·examFiguresMath.ts 본문)을 읽지 마라.** 필요한 정보는 전부
여기에 있다. 이 브리프에 없는 규칙을 임의로 만들지 말고, 애매하면 보고만 남겨라.

## 0. 작업 계약 (위반 즉시 실격)

- 담당 파일 **3개만** 생성한다(자기 몫 섹션 참조). 그 외 파일 수정·생성·실행 절대 금지
  (tsc·node·스크립트 실행 금지 — 검사는 메인 세션이 한다).
- **출력 분할 절대 규칙**: 응답(턴)당 도구 호출 1개, 호출당 문항 5개 이하.
  파일당 Write 1회(헤더+첫 5문항) 후 Edit로 5문항씩 이어 붙인다(마지막 회차만 2~3문항).
  Edit는 파일 끝의 `];` 한 줄을 "새 문항들 + `];`"로 교체하는 방식.
  **전체 설계를 먼저 끝내려 하지 마라** — "다음 5문항만 설계 → 즉시 저작"의 리듬을 반복한다.
  (전체 설계 선행 에이전트는 1시간 무출력 후 죽는다 — 실사고 2회.)
- 완료 보고에는 파일별 `grep -c "id: \"m2u6e" <파일>` 결과(문항 수)를 포함한다. 이것만은 Bash 허용.
- 발견한 문제(규칙 충돌·조달 불가)는 고치려 들지 말고 보고만.

## 2. ExamItem 계약 요약

```ts
{
  id: string;            // "m2u6e001" ~ "m2u6e200"
  lessonId: string;      // const L
  type: "mcq" | "multi" | "num" | "word";
  prompt: string;        // HTML 허용. num은 단위·표기 지시를 문두에 명시
  bogi?: string[];       // ㄱㄴㄷ 합답형 보기 상자(라벨 ㄱ.ㄴ.ㄷ.은 렌더러가 붙임 — 문장만)
  figure?: string;       // SVG 문자열(그림 헬퍼 호출). figureDark 금지(전부 밝은 그림)
  options?: string[];    // mcq 5지 · multi 4~5지
  answer: number | number[] | string;  // mcq 저작 인덱스 · multi 인덱스 배열 · num 문자열 · word 칩
  shuffle?: boolean;     // 라벨형 보기(ㄱㄴㄷ 조합·①~⑤ 지칭)만 false — false면 정답을 첫 보기에 두지 않는다
  numKind?: "int" | "dec" | "frac";
  unitLabel?: string;    // num 입력 칸 옆 단위(경우의 수는 "가지", 확률·소수는 생략)
  bank?: string[];       // word 칩 8~10개, bank[0]=정답으로 저작(표시는 자동 셔플)
  diff?: 1 | 2 | 3;      // 전 문항 필수
  explain: string;       // 250~450자 해요체(아래 규격)
  core: string;          // 한 줄 요약(레슨 바로가기 옆 문구)
}
```

- **mcq는 5지 선다**가 기본. multi는 "옳은 것을 모두 고르세요" 4~5지(정답 2~3개).
- 셔플: 일반 보기는 자동 표시 셔플이므로 정답 인덱스는 자유. `shuffle: false`는 라벨형
  (ㄱㄴㄷ 조합 보기·①~⑤ 그림 지칭)만 쓰고, 이때 **정답을 options[0]에 두지 않는다**.
- bogi 합답형: `bogi: ["문장1", "문장2", "문장3"]` + `options: ["ㄱ", "ㄴ", "ㄱ, ㄷ", "ㄴ, ㄷ", "ㄱ, ㄴ, ㄷ"]`
  + `shuffle: false`. 해설에서 ㄱㄴㄷ 라벨 지칭은 허용(고정 라벨).
- **문항은 자기완결**: "앞 문제에서"류 이웃 참조 금지(랜덤 추출). 그림도 문항 각자 소유.

## 3. 해설(explain) 규격 — 문제집 최상급

- 형식: `"<span class='xh'>정답 풀이</span>...<span class='xh'>오답 하나씩 격파</span>..."`
  (num·word는 둘째 소제목을 `이렇게 틀려요`로). 250~450자, 해요체, "틀렸어요"로 끝내기 금지.
- 정답 풀이의 계산은 한 줄씩 `①②③` + `<br>` 로 끊는다.
- 오답 격파는 **보기 위치(N번째) 지칭 금지, 내용 인용으로**. 오답이 나오는 경로를 지어내지
  마라 — 재현되는 실수만 서술하고, 확실치 않으면 "OO을 대입/나열해 보면 어긋나요" 검증형으로.
- 해설 속 분수도 mfmt로: `"...확률은 " + mfmt("{5/36}") + "이에요."` 꼴의 문자열 연결.
- core: 한 문장 구호(예: "두 주사위는 6×6=36칸 표로!").

## 4. 언어 규칙 (기계 검사 대상 — 위반 즉시 실격)

- **금지어**: 여사건, 시행, 수형도, 순열, 조합(일상어 "조합"도 회피 — "짝"·"경우"로),
  nPr, nCr, 조건부 확률, 독립(→"서로 영향을 끼치지 않는다"), 수학적 확률, 통계적 확률,
  기댓값, 배반, 표본, 복원/비복원(용어 자체 금지 — 서술로 "다시 넣은 후"만).
  '시행'은 "실험이나 관찰"로, '여사건'은 "일어나지 않을 확률"로, '수형도'는 "나뭇가지 그림"으로.
- **합·곱 조건 서술 통일(전 문항·해설)**: 덧셈 조건 = "동시에 일어나지 않을 때",
  곱셈 조건 = "서로 영향을 끼치지 않을 때". 이형 표현(겹치지 않을 때, 무관할 때,
  영향을 주지 않을 때, 미치지 않을 때 등) 금지 — grep으로 검출된다.
- **연속 뽑기는 반드시 "꺼내 확인하고 다시 넣은 후"(복원)만**. 뽑고 안 돌려놓는 상황을
  확률 곱셈에 쓰지 마라(다음 학년). 단, "여러 장 중 2장을 동시에 뽑는" 전수 나열 세기
  문항(경우의 수·기본 확률)은 허용 — 곱셈 규칙 적용 대상만 아니면 된다.
  곱셈 소재는 복원 뽑기·서로 다른 장치(주사위 두 번, 두 원판, 동전+주사위, 명중, 자유투)로.
- 허용 어휘: 사건, 경우의 수, 확률, 일어나지 않을 확률, "적어도", 임의로, 나뭇가지 그림,
  상대도수(중1 정식 어휘). 확률 공식 조건 "가능성이 모두 같을 때"와 "임의로"를 문두에 챙긴다.
- **em대시(—) 금지**(제목·본문 전부 — 콜론·쉼표로). 뺄셈·음수 기호는 **U+2212(−)**,
  단 num answer 저장값만 ASCII. 이모지 금지. 해요체.
- 변수 확률 p·q는 `<i class='mv'>p</i>` 또는 mfmt("0≤p≤1")(소문자는 자동 이탤릭).
- **mfmt 규약**: 분수 `mfmt("{5/36}")`, 부등호 `mfmt("x>9")`·`mfmt("0≤p≤1")` OK.
  **분수 분자는 하이픈만 지원**((7x−5)/4 꼴 OK, (3x+1)/2는 그룹 렌더) — 이 단원 분수는
  전부 숫자 분수라 무관하지만 {1/2}+{1/3} 같은 식 전체를 한 mfmt에 넣지 말고 조각내 연결.
- raw HTML 문자열에서 `<` 뒤에 영문자가 오면 태그로 파싱된다(`a<b` 금지 — `p<1`처럼 뒤가
  숫자·기호면 안전, 불안하면 mfmt를 써라. mfmt는 자동 이스케이프).
- 실체 없는 장비·조어 금지(센서·드론·비콘·측정 장치·"~식" 가짜 명사). 소재는 중2가 일상에서
  아는 것만: 주사위·동전·카드·구슬·제비·원판·급식·문구점·버스·날씨·경기·게임 등.
  순수 계산 문항은 무맥락 서술("서로 다른 두 개의 주사위를...")이 정답이고 억지 맥락 금지.

## 5. num 규격

- L1~L3(경우의 수): `numKind: "int"`, `unitLabel: "가지"`, 문두 "~경우의 수를 구하세요".
- L4~L8(확률): `numKind: "frac"`, **answer는 기약분수 ASCII 문자열 "a/b"**, unitLabel 생략,
  문두에 **"기약분수로 구하세요"** 반드시 명시. (채점은 값 동치라 3/6 입력도 정답 처리되지만
  answer 저장은 기약이 표준.) 확률이 0 또는 1이 되는 답은 `numKind: "int"`에 answer "0"/"1"
  (L5 성질 문항 전용, 최대 2문항).
- L9(활용): `numKind: "dec"`, answer "0.05" 꼴 문자열, unitLabel 생략, 문두 "소수로 구하세요".
- **num 정답은 파일 안에서 중복 금지**, 파일 사이에서도 가급적 분산(자기 몫의 수치 대역 참조).
- num에도 진짜 사고를 요구하라: 단순 대입만 7개 나열 금지 — 조건 세기, 역산("확률이 1/4이
  되려면 당첨 제비는 몇 장?" → int), 규칙 결합(1−p 후 곱) 섞기.

## 6. word 규격

- `bank` 8~10칩, bank[0]=정답. 빈칸 문장(`___`)이 있는 정의·규칙 문장 프롬프트.
- **빈칸 문장을 참으로 만드는 칩은 정답 하나뿐이어야 한다**: 정답의 상위 분류 칩 금지
  (예: '경우의 수' 답이면 '수'·'가짓수' 계열 위험 — 문장에 대입해 참이 되는지 전수 확인),
  동의어·포함 관계 칩 금지.
- **'확률'이 답인 문장에 '상대도수' 칩 금지, '상대도수'가 답인 문장에 '확률' 칩 금지**
  (극한 서사라 문장을 참으로 만들 수 있음 — 정의 문장을 배타적으로 쓸 것).
- "더한다"↔"곱한다" 같은 배타 쌍은 같은 bank 허용(좋은 오답). 통계 어휘(변량·도수·계급·
  대푯값)는 오답 칩으로 적극 활용(중1 기왕).
- word 정답 20개는 아래 할당표 고정 — 다른 용어로 바꾸지 마라. 파일 간 정답 중복 없음.

| 레슨 | word 정답 |
|---|---|
| L1 | 사건 · 경우의 수 |
| L2 | 더한다 · 겹침 |
| L3 | 곱한다 · 나뭇가지 그림 |
| L4 | 임의로 · 반반 |
| L5 | 1 · 0 |
| L6 | 적어도 · 모두 |
| L7 | 동시에 · 공평 |
| L8 | 영향 · 작아진다 · 곱셈 |
| L9 | 확률 · 상대도수 · 건수 |

- 주의: L3 '곱한다'와 L8 '곱셈'은 다른 파일이지만, 각 bank에 상대 칩을 넣지 마라(문장 참 위험).
  L9 '확률' 문장은 "많은 횟수의 실험에서 상대도수가 가까워지는 일정한 값" 각도(극한),
  '상대도수' 문장은 "(사건이 일어난 횟수)÷(전체 실험 횟수)로 구한 값" 각도(유한 기록)로 분리.
- word 프롬프트의 정의문이 같은 단원 mcq 정의형 문항과 같은 용어를 다루지 않게 하라
  (예: L1 word가 '사건' 정의라면 L1 mcq에 '사건의 뜻' 문항 금지).

## 7. 그림 규격 (시그니처 고정 — 임의 변경 금지)

전부 SVG 헬퍼 호출로 `figure:`에 넣는다. **레슨 그림·아래 "사용 금지" 헬퍼는 절대 쓰지 마라.**
사용 금지: diceEventFig, orGroupsFig, branchFig, convergeFig, coinCasesFig, areaModelFig,
flightTableFig (경우의 수·곱·공식이 그림에 인쇄돼 유출).

허용 10종과 호출 예:

```ts
// ① 순서쌍 표 — 반드시 빈 상태(pick=()=>false)로만. 강조 칸 금지(세기 과제 유출).
figure: pairGridFig(() => false, "두 주사위의 모든 경우"),
// ② 확률 눈금(0~1 수직선 + 마커) — at은 0~1 실수(실제 값 위치 비례), label은 ㉠㉡ 기호 위주
figure: probLineFig([{ at: 0.25, label: "㉠", tone: "blue" }, { at: 1, label: "㉡", tone: "green" }]),
// ③ 칸 그리드(전체 total칸 중 part칸 사건) — 라벨에 확률값 인쇄 금지(칸 수는 조건 데이터라 OK)
figure: restGridFig(12, 5, "당첨", "꽝"),
// ④ 균등 원판(total 등분, wins 인덱스 당첨) — "당첨 N칸/꽝 M칸/전체 N등분" 배지는 조건 제시라 OK
figure: spinnerFig(10, [1, 4, 6], "별"),
// ⑤ 나뭇가지 시험판 — 가지 수 = 실제 경우와 정확히 일치(재사용 금지는 second 배열로 직접 반영)
figure: m2ExamBranchFig({ head1: "십의 자리", head2: "일의 자리",
  first: ["1", "3", "5"], second: [["3", "5"], ["1", "5"], ["1", "3"]] }),
// ⑤-2 fold형(일부만 그린 그림 — "나머지는 같은 규칙" 문두 서술과 세트)
figure: m2ExamBranchFig({ head1: "모자", head2: "옷", first: ["가", "나", "다"],
  second: [["티", "후드"], [], []], fold: [1, 2] }),
// ⑥ 불균등 원판(중심각 실각 렌더 — deg 합이 반드시 360, 1/4 칸이면 90) — 칸 수 vs 넓이 함정 전용
figure: m2ExamSpinnerFig({ slices: [{ deg: 180, label: "당첨", win: true },
```
