# u5l1 「힘의 표현」 레슨 스펙 (파일럿)

원 교과서: 대단원 V 힘의 작용 > 중단원 1 여러 가지 힘 > 소단원 1 힘의 표현 (책 156~157쪽).
전사 근거: unit5_notes_L1.md(156~157쪽), 평가 근거: unit5_notes_assess.md의 [표현] 태그 문항.
표준 공식(LESSON_GUIDE.md): 도입(hook) → 랩(2개) → recap → 문제(5문, 능동형 먼저).

## 교과 핵심 (이 레슨이 가르치는 전부)
- **힘** = 물체의 **모양**이나 **운동 상태**를 변하게 하는 원인. (운동 상태 = 빠르기 + 운동 방향)
- 힘의 효과 3갈래: 모양 변화 / 운동 상태 변화(정지→움직임·움직임→정지·빠르기·방향) / 둘 다.
- **화살표 표시 3요소**: 작용점(화살표 시작) · 방향(화살촉) · 크기(길이 비례).
- 힘의 단위 **N(뉴턴)**.
- 힘의 평형·합력·중력은 **다음 레슨** — 여기서 다루지 않는다.

## 스텝 시퀀스 (9스텝)

### 1. hook — scene "balloon" (신규 장면, steps/hook.ts에 추가)
- 교과서 도입: "풍선에 힘이 작용하면 어떻게 될까?" (156쪽).
- 무대: 헬륨 풍선이 줄에 매달려 둥실 떠 있음(살짝 흔들리는 idle). 래스터 `public/hooks/balloon.png`
  시도 → 실패 시 SVG 폴백(sceneArt 관례).
- 미세 상호작용: **줄(또는 풍선)을 아래로 드래그** → 풍선이 끌려 내려오고 줄이 팽팽해짐.
  임계(40px+) 넘게 당긴 뒤 **놓으면** 스프링 복귀(오버슈트 둥실). 이 왕복 1회로 finish.
- 얼굴: 당기는 중 "surprised", 복귀 관찰 "curious", finish 후 900ms "smile"(hook 공통).
- 헬퍼 실시간: 대기 "풍선 줄을 잡고 아래로 당겨 보세요" → 당기는 중 "당기는 동안 풍선이 내려와요 —
  손을 떼면?" → 복귀 후 완료 문구.
- 채점 없음(예측 아님). CTA는 finish 시 enableCTA.

### 2. forceEffects — 힘의 효과 관찰 랩 (신규 렌더러 steps/forceEffects.ts)
- 교과서 156쪽 해보기(캔·골프공·야구공·배구공) + 157쪽 그림 V-1을 하나의 무대로.
- 다크 무대(.stage) + canvas(fitCanvas/createLoop). 상단 세그(.seg) 4버튼: 캔 / 골프공 / 야구공 / 배구공.
- 각 장면: 무대 탭(또는 하단 "힘 주기" 게이지 버튼)으로 힘을 가함 →
  - **캔**: 손이 쥐어 찌그러짐 → 배지 "모양이 변했어요"
  - **골프공**: 정지한 공 → 퍼터가 침 → 굴러감 → 배지 "정지 → 움직임"
  - **야구공**: 날아오는 공(루프) → 글러브로 받음 → 멈춤 → 배지 "움직임 → 정지"
  - **배구공**: 떨어지는 공 → 손이 받아침 → 경로가 꺾여 위로 → 배지 "방향이 바뀌었어요"
- 캔버스 상단 HUD: 수집 필 4칸(모양/출발/정지/방향) — 장면 실행마다 체크 점등 + HAPTIC.correct.
- 4/4 수집 → enableCTA + recordQuiz(true) (관찰 랩 규칙).
- 소품 재질: ui/labProps 문법(그라데이션·접촉 그림자·윗면 하이라이트). 공은 radial 그라데이션 + 이동 잔상.
- 배지·라벨 문구는 unit5.ts 데이터가 아닌 렌더러 상수여도 됨(장면 고정형 랩).

### 3. forceArrow — 핵심 랩: 화살표 미션 (신규 렌더러 steps/forceArrow.ts)
- 다크 무대 + 격자(1칸 = 5 N, 우하단 범례 "1칸 = 5 N"). 나무 상자(labProps 재질) 무대 중앙 바닥.
- **미션 1**: "상자를 오른쪽으로 미는 힘 30 N를 화살표로 나타내요."
  - 상자 위 pointerdown(작용점은 상자 중심에 스냅, 흰 링+빨강 코어 도트) → 드래그 방향·길이로
    화살표 프리뷰(빨강 발광, 5 N 스냅) + HUD 실시간 "지금 힘: 25 N".
  - 릴리즈 판정: 수평 주방향 && 목표 N 일치 → 성공(정답 햅틱 + 상자가 힘 방향으로 슝 밀림).
    틀리면 스낵("조금 더 길게!", "방향이 반대예요", "너무 비스듬해요") + 재시도.
- **미션 2**: "이번엔 왼쪽으로 15 N!" — 절반 길이 체감. 성공 시 상자 왼쪽으로 절반 거리 이동.
- **미션 3 (확인·채점)**: 상자 2개 등장 — A: 오른쪽 20 N 화살표, B: 왼쪽 40 N 화살표(색 동일, 길이만 2배).
  "더 **센** 힘을 받는 상자는?" → 상자 탭 선택. **첫 시도를 recordQuiz**로 집계(정답 B).
  정답/오답 모두 헬퍼로 근거 짚기("길이가 크기예요 — 방향은 세기와 무관!") 후 enableCTA.
- 좌표·물리는 순수 캔버스(rAF), stopAllLoops 대상 createLoop 사용. cleanup 반환.

### 4. recap — 카드 3장 + note
- narrator: 방금 체험 회수(무대에서 화살표 그린 것).
- 카드(색·미니아트, ui/forceFigures.ts miniArt):
  1) **힘** `#0CA6C0` art "force" — 정의(모양·운동 상태 변화의 원인) + 단위 N. more: 뉴턴 이야기 등.
  2) **힘의 효과** `#F0705E` art "effects" — 모양/운동 상태(빠르기·방향)/둘 다. more: 야구공 스윙 = 둘 다.
  3) **화살표 3요소** `#7C6BFF` art "arrow3" — 작용점·방향·크기(길이 비례). more: 설계도·일기예보 바람 화살표 등.
- note(bulb/amber): "미는 것만 힘이 아니에요 — 당기고, 받치고, 쥐는 것도 전부 힘" 류.
- 카피는 Opus 초안 → 리라이트.

### 5~9. 문제 (binSort → hotspot → mcq → mcq(figure) → ox)
- **5 binSort** (제목형): "힘의 효과 분류하기". 통 3개: 모양 변화(#F0705E) / 운동 상태 변화(#0CA6C0) / 둘 다(#7C6BFF).
  칩 6: 찰흙 누르기(모양) · 알루미늄 캔 쥐기(모양) · 정지한 골프공 퍼팅(운동) · 굴러오는 공 멈춰 세우기(운동) ·
  방망이로 야구공 치기(둘 다) · 고무줄 길게 당기기(모양).
- **6 hotspot find** (제목형): 그림 V-2 재현 `forceFigures.arrowAnatomy()` + `arrowAnatomySpots`
  (spots 순서 = 질문 순서: 힘의 작용점 → 힘의 방향 → 힘의 크기). dark 아님(밝은 그림).
- **7 mcq n1/3**: 대단원 마무리 1번 변형 — "힘이 작용할 때 나타나는 현상이 **아닌** 것은?"
  ①모양 변화 ②움직이던 물체가 멈춤 ③정지한 물체가 움직임 ④**물체를 이루는 물질의 종류가 변함**(정답) ⑤운동 방향 변화.
- **8 mcq n2/3 (figure)**: `forceFigures.arrowCompare()` — (가) 오른쪽 화살표, (나) 같은 방향 2배 길이.
  "옳은 것은?" 정답: (나)의 힘의 크기가 (가)의 2배. 오개념 선지: 방향이 더 크다 / 더 오래 작용한다 /
  크기는 같다 / 작용점이 더 크다.
- **9 ox n3/3**: "화살표의 **길이**는 힘의 **방향**을 나타낸다" → X. 해설: 길이=크기(비례), 방향=화살촉.
