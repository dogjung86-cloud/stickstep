# 중2 과학 V 식물과 에너지 이미지 재생성 기록

기존 10개 WebP의 **용도와 생성 의도 요약**과, 이후 추가한 광합성 랩 식물 에셋의 실제 생성 프롬프트를 기록한다. 기존 이미지 생성 당시의 실제 프롬프트 원문은 이 저장소에 남아 있지 않으므로, 아래 기존 항목의 문장은 원문을 복원하거나 새 프롬프트라고 주장하지 않는다.

## 공통 재생성 원칙

- 이미지 안에는 글자, 숫자, 수식, 화살표, 라벨을 넣지 않는다. 정확한 과학 정보는 HTML·SVG·Canvas 오버레이로 얹는다.
- 인물 컷은 기존 앱의 고급 스틱맨 만화 문법을 따르고, 장면의 핵심 행동과 표정이 작은 화면에서도 즉시 읽혀야 한다.
- 실사풍 그림은 깨끗한 조명, 명확한 주 피사체, 넉넉한 여백을 유지한다. 기공·엽록체·물관·체관처럼 판정이 필요한 구조를 생성 이미지에 맡기지 않는다.
- 최종 파일은 WebP로 만들고 긴 변 960px 이하를 지킨다. 품질 68~74를 출발점으로 하되 경계와 얼굴이 무너지지 않는 선에서 압축한다.
- 장당 120KB 아래를 우선하며, 현재 묶음처럼 30~75KB 수준을 목표로 한다. 투명도가 필요하지 않으면 불필요한 알파 채널을 제거한다.

## 광합성 상호작용 랩 식물 에셋

`public/plant/labs/leaf-factory-plant.webp`는 `leafFactoryLab`의 잎·줄기·뿌리 베이스다. 생성 이미지에는 구조 라벨과 이동 화살표를 넣지 않고, 버튼을 누르면 베이스의 불투명도를 낮춘 뒤 코드 오버레이로 빛·이산화 탄소·물의 이동을 보여 준다.

실제 생성 프롬프트의 핵심 조건은 다음과 같다.

- 중학생 광합성 상호작용용 건강한 어린 활엽 식물 한 그루, 곧은 줄기와 넓은 잎 2~3장, 갈라진 뿌리 전체를 세로 구도로 표현한다.
- 꽃·열매·화분·흙·글자·숫자·라벨·화살표·아이콘·도해 콜아웃은 넣지 않는다.
- 프리미엄 교육용 보태니컬 일러스트 질감, 작은 모바일 화면에서도 잎·줄기·뿌리가 구분되는 실루엣을 유지한다.
- 초록 식물과 분리되는 단색 `#ff00ff` 크로마키 배경으로 생성하고, 로컬 후처리로 알파를 만든다.
- 최종 에셋은 긴 변 960px의 투명 WebP로 저장한다.

### 잎 중심 반응 모형 v2

`public/plant/labs/leaf-factory-diagram-v2.webp`는 참고 도해의 구도만 계승해 새로 생성한 잎·줄기 베이스다. `imagegen` 기본 내장 경로로 생성한 뒤 `#ff00ff` 크로마키를 제거하고, 긴 변 960px 투명 WebP로 가공한다. 참고 이미지는 구도 참고용이며 원본의 글자·화살표·수식·입자·아이콘·그림 자체를 복제하지 않는다.

실제 생성 프롬프트의 핵심 조건은 다음과 같다.

- 짧은 곧은 줄기가 왼쪽 아래에서 올라오고, 넓은 초록 잎 한 장이 오른쪽으로 펼쳐지는 정사각 구도다.
- 잎의 중앙 맥과 잔맥은 자연스럽게 보이되, 중앙에는 앱이 반응 상자와 입자를 얹을 수 있는 비교적 차분한 공간을 둔다.
- 프리미엄 반실사 교육용 보태니컬 일러스트로 만들고, 잎과 줄기의 연결은 과학적으로 자연스럽게 표현한다.
- 글자·숫자·라벨·화살표·입자·태양·반응 상자·수식·워터마크는 넣지 않는다.
- 생성 배경은 균일한 `#ff00ff`이며, 식물에는 마젠타나 분홍색을 사용하지 않는다.

## 2026-07-13 보정·추가 에셋

### `public/plant/cuts/g2u5l6.webp`

기존 컷을 `imagegen` 기본 내장 편집 경로로 보정했다. 뿌리는 줄기 중간이 아니라 식물 맨 아래에서만 자라고, 잎은 공중에 떠 있지 않고 줄기와 곁가지에 자연스럽게 붙도록 했다. 토마토 열매는 위쪽 곁가지에 달고, 자전거 배달원은 실제로 이어진 줄기 위를 달리게 했다. 글자·라벨·화살표·워터마크는 생성 이미지에 넣지 않았다.

### 물의 이동·양분의 이동 토글 그림

한 장에 두 식물을 나란히 그린 `public/plant/figs/xylem-phloem-transport.webp`는 방향 오류를 확실히 막기 어려워 사용을 중단했다. 같은 단원에서 쓰는 글자와 화살표가 없는 `public/plant/figs/whole-plant.webp`를 두 토글의 공통 베이스로 재사용하고, 이동 경로는 앱 코드의 SVG 오버레이로 분리했다. 물의 이동은 모든 화살표 끝점의 세로 좌표가 시작점보다 위에 있도록 고정해 위·위쪽 왼쪽·위쪽 오른쪽 방향만 허용한다. 양분의 이동은 잎에서 어린순·열매·뿌리로 가므로 위쪽과 아래쪽 경로를 모두 보여 준다.

### `public/plant/figs/day-observatory.webp` · `night-observatory.webp`

기존 참고 도해와 구도가 닮지 않도록 참고 이미지를 입력으로 사용하지 않고 `imagegen` 기본 내장 경로에서 낮 장면을 새로 생성했다. 야외의 큰 나무를 좌우로 나누는 구성 대신, 학교 식물 관찰실의 원형 관찰대 위 화분 식물을 한 장면씩 크게 보여 주는 구도다. 이어서 낮 장면을 편집 대상으로 사용해 식물·화분·시점·실내 배치는 유지하고 조명만 달빛이 비치는 밤으로 바꿨다.

실제 생성 프롬프트의 핵심 조건은 다음과 같다.

- 낮은 곡선형 창으로 햇빛이 들어오는 밝은 식물 관찰실, 밤은 같은 장소에 초승달과 푸른 달빛이 보이는 장면이다.
- 같은 콩 식물과 화분, 관찰대, 카메라 시점을 유지해 토글 전환 시 시간대만 달라졌다고 읽히게 한다.
- 기존 도해의 두 칸 나무 비교, 큰 야외 나무, 좌우 화살표 배치를 사용하지 않는다.
- 글자·숫자·라벨·화살표·기체 입자·수식·도해 상자·워터마크는 생성 이미지에 넣지 않는다.
- 낮·밤 버튼, 광합성·호흡 상태, 기체 이동 방향과 설명은 앱의 고대비 HTML/CSS 레이어가 담당한다.
