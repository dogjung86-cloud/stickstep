# LESSON_GUIDE — 표준 레슨 공식 (3단원에서 확정)

3단원(열)을 기준으로 확정한 레슨의 표준 형태다. **새 단원은 처음부터 이 공식으로 만들고,
단원 I·II는 아래 마이그레이션 체크리스트대로 고친다.** (설계 헌법은 CLAUDE.md, 이 문서는 실행 기준.)

## 표준 레슨 공식 — "도입 → 랩 → 정리 → 문제"

모든 레슨은 이 네 박자를 지킨다. 개념 하나 → 바로 퀴즈 금지(개념을 다 가르친 뒤 문제).

### 0) 스틱맨 쌤은 어디서나 같은 얼굴
- 아바타는 반드시 `ui/avatar.ts`의 `stickAvatar()`를 쓴다(만화와 같은 손그림 래스터,
  `public/comics/avatar/0..4.png`, 실패 시 SVG 폴백). 벡터 `stickmanHead()`를 직접 쓰지 말 것.
- 표정 교체는 `setStickAvatar(el, "surprised" | "curious" | "smile")` — hook의 미세 상호작용에
  반응하면 생동감이 산다(컵을 만지면 놀란 얼굴 등).

### 1) 도입 — 스틱맨 쌤이 연다 (`hook` 또는 `comic`)
- **모든 레슨의 첫 스텝에는 스틱맨 쌤이 등장한다.** 두 가지 형태:
  - `hook` 스텝: 스틱맨 말풍선 + **미세 상호작용**(만져 보기·넣어 보기·예측 고르기).
    긴 이야기가 필요 없을 때. 상호작용을 마쳐야 CTA가 열린다.
    - 장면은 `steps/hook.ts`에 추가한다(현재: `cups` 두 컵 만지기, `egg` 달걀 퐁당+예측,
      `beach` 모래/바닷물 만지기, `wire` 여름/겨울 전깃줄 토글+예측,
      `smell` 급식 뚜껑 열기+예측, `juice` 얼음/주스 줍기, `wrap` 배달 랩 부풀기+예측,
      `ramen` 끓는 냄비 화력 최대+예측,
      V 힘: `balloon` 풍선 끌었다 놓기, `tugrope` 멈춘 줄다리기+예측, `bow` 활쏘기,
      `iceslip` 빙판vs자갈길+예측, `bottle` 페트병 밀어넣기, `rollstop` 멈추는 공+예측,
      VI 기체: `polar` 북극곰 얼음판+예측, `bubblewrap` 뽁뽁이, `foilballoon` 은박 풍선 안↔밖,
      `pingpong` 찌그러진 탁구공 구출+예측,
      VII 태양계: `stargaze` 밝은 점 망원경, `planetsize` 지구 11개=목성+예측,
      `shadowclock` 시계탑 그림자+예측, `moonpic` 달 사진 비교+예측, `sunglasses` 관측 안경+예측,
      중2 III 빛과 파동(hookLight.ts): `mirrortown` 산 위 거울 마을 비추기, `coinmagic` 물 부으면 동전+예측,
      `darkroom` 전등 켜기+오개념 예측, `catmirror` 거울 고양이+예측, `spoon` 숟가락 앞뒤 토글,
      `pointillism` 점묘화 확대+예측, `fishing` 낚시찌 물결+예측, `kalimba` 칼림바 두 막대(실제 소리)+예측).
    - **예측(prediction)은 채점하지 않는다** — 정답을 알려주지 않고 "실험으로 확인해 봐요"로 랩에 연결.
  - `comic` 스텝: 서사가 필요할 때(과학사, 캠핑 이야기 등). 컷 이미지는 codex 발주(CLAUDE.md 참고).
- 도입의 목적은 지식 전달이 아니라 **궁금증 생성**. 개념 설명은 랩과 recap에 맡긴다.

### 2) 랩 — 핵심 상호작용 (1~3개)
- 다크 무대(`.stage`) + 발광 비주얼. 조작 목표를 달성하면 `enableCTA`(팝+햅틱)로 해제.
- **예측→실행→확인** 구조를 우선한다(전도 레이스, 난방기 배치처럼). 예측이 있는 랩은
  `recordQuiz(예측 정답 여부)`로 채점에 반영, 관찰만 하는 랩은 완료 시 `recordQuiz(true)`.
- helper 한 줄이 실시간으로 "지금 무엇을 보고 있는지"를 말로 짚어 준다.
- **3D 랩(VII 기준 구현)**: 명암·정렬이 개념 그 자체인 주제(달 위상, 일식·월식)는 three.js를 쓴다.
  `ui/space3d.ts`는 반드시 `await import(...)`로 동적 로드(초기 번들 분리),
  가로로 긴 장면(태양계 일렬·일식 정렬)은 `ui/rotateStage.ts`(90° 회전 오버레이 + `mapPoint` 포인터 리매핑)로.
  움직이거나 작은 3D 탭 대상엔 투명 히트 프록시(혜성처럼)를 붙인다. 프레임마다 `st.render()` 호출을 잊지 말 것.

### 3) 정리 — `recap` 스텝으로 통일
- **정리 자리는 recap 하나로 통일한다. 표(table)·긴 문단 나열 금지.**
- 구성: 스틱맨 말풍선 → 개념 카드(미니 일러스트 + 굵은 핵심 한두 문장 + 예시 칩) → (선택) 콜아웃 → 스틱맨 아웃트로.
- 카드는 2~4장. 카드 색은 개념 정체성 색(예: 전도 `#FF6B4A`, 대류 `#0CA6C0`, 복사 `#F04452`).
- 미니 일러스트는 단원별 모듈에 64×64 SVG로 추가(열: `ui/heatFigures.ts`, 물질: `ui/matterFigures.ts`의 `miniArt()`,
  기체: `ui/gasFigures.ts`의 `gasMiniArt()`, 태양계: `ui/spaceFigures.ts`의 `spaceMiniArt()`).

### 4) 문제 — 형식을 섞는다 (3~6문)
- 순서: **능동형 먼저**(binSort·order), 그다음 mcq/multi/ox. 그림 문제 1개 이상 권장.
- 그림은 손코딩 SVG(`ui/*Figures.ts`). **색이 정답의 단서가 되지 않게** 설계(예: 입자 상자 3개는 전부 같은 색).
- 오답 해설은 반드시 실제 오개념을 짚는다. 보기에도 대표 오개념을 심는다
  (예: "입자가 직접 이동하며 전도한다", "입자가 완전히 멈춘다").
- `n/of` 번호는 mcq·ox·multi에만 붙인다(binSort·order는 제목형).

## 공용 규칙 (이미 전 단원에 적용된 것)
- **뒤로 가기**: 플레이어 헤더 왼쪽에 이전 스텝 버튼(첫 스텝에선 숨김). 채점은 **스텝당 첫 시도만**
  집계된다(`player.ts`의 recordedSteps) — 뒤로 갔다 다시 풀어도 정확도가 부풀지 않는다.
- **binSort는 드래그 앤 드롭이 기본**(칩을 끌어 통 위에 놓기). 탭(칩 탭→통 탭)도 함께 동작한다.
  통에 담긴 칩은 다시 끌거나 탭해서 옮기거나 트레이로 되돌릴 수 있다.
- **단원 탭은 마우스 드래그로도 스크롤**된다(터치는 네이티브).
- 상호작용 시뮬은 "가짜 연출"보다 **원리가 보이는 모형**을 쓴다(예: 열평형 랩은 입자마다 온도를 갖고
  충돌로 열을 주고받는 운동론 모형 — 칸막이 제거 → 빠른 입자는 느려지고 느린 입자는 빨라짐).

## 문구 규칙
- 완전 한글 해요체, 이모지 금지(SVG 픽토그램만), 중1은 '분자'가 아니라 '입자'.
- **교과서 출판사 이름(예: 미래엔)을 UI 문구에 절대 쓰지 않는다.** 단원 배너 표기는
  "2022 개정 교육과정", 레슨 출처는 `standard: "책 NN~NN쪽"`만.
- 랩·문제 해설에서 방금 한 체험을 되짚는다("무대에서 슬라이더를 올렸을 때…").

## 단원 I·II 마이그레이션 체크리스트
새 세션에서 이 문서를 읽고 아래를 레슨 단위로 적용한다.
**단원 I은 마이그레이션 완료(2026-07-07)** — L1 만화+recap, L2 colorcups 훅+recap,
L3 「과학이 바꾼 세상」 만화(comics/u1l3)+타임라인 order+recap, L4 speaker 훅+techCards+recap,
L5 smokestack 훅+binSort(8아이템)+recap. 대단원 마무리 레슨은 제거(사용자 결정 — 마무리 없이 5레슨).
훅 장면은 steps/hookCiv.ts, recap 미니아트는 ui/civFigures.ts.
**단원 II도 마이그레이션 완료(2026-07-07)** — 6레슨 전부 hook 도입 + 기존 시그니처 랩(세포 핫스팟·현미경·구성 단계·핀치 자연선택·검색표) 유지 + recap 카드 + 문제. 훅 6종 = steps/hookBio.ts(cellzoom·stain·bodycount·ladybugs·batbird·foodweb), recap 미니아트 = ui/bioFigures.ts. curio 2개(코끼리 세포=L3 orgLevels, 바이러스=L5 dichotomKey — 두 렌더러에 curio prop 추가). 마지막 축하 concept 스텝 제거. **단원 I·II 모두 표준화 끝.**

1. **도입 통일** — 각 레슨 첫 스텝을 확인:
   - 스틱맨 없이 `concept`로 시작하는 레슨 → `hook`(새 장면 필요 시 `steps/hook.ts`에 추가) 또는
     기존 `comic`으로 교체. 도입 concept의 설명 문장은 hook 말풍선/recap 카드로 이관.
   - u1l1(린드 만화)·u3l3(캠핑 만화)는 이미 표준 충족.
2. **정리 통일** — `table`·"핵심 정리" concept 스텝을 `recap`으로 교체:
   - u1l1의 "탐구 6단계, 한눈에" table → recap 카드 6장(또는 3+3).
   - 각 단원 미니 일러스트 모듈 신설(u1: 탐구 아이콘, u2: 생물 아이콘 — `figures.ts` 아트 재활용 가능).
3. **문제 점검** — binSort가 있는 레슨은 드래그로 자동 개선됨(코드 수정 불필요). 그림 문제의
   축 라벨 겹침·색 단서 여부 점검.
4. **문구 스캔** — `grep 미래엔` 0건 확인(완료됨), 해요체·이모지 점검.
5. **검증** — 프리뷰에서 레슨 E2E(도입 상호작용 → 랩 목표 → recap → 전 문제 정답/오답 시트)
   + 뒤로 가기 왕복 + 드래그 분류 확인.
