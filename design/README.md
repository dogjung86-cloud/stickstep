## 확정된 방향 (사용자 확정 2026-07-12)

1차 개편 묶음 = **A+B 격상판**:
- **발자국 트레일**: 걸어온 구간의 연결선을 잉크 발자국으로(갈 길은 옅게).
  serpentine 경로 좌표를 그대로 샘플링해 도장 — 목업의 stamp() 함수가 구현 참고.
- **스틱맨 워커**: 현재 노드 위에 내 아바타 스틱맨(깃발). 레슨 완료 귀환 시 다음 노드까지
  걷기(8fps 두 포즈 플립북, 스플래시 125ms 리듬) + 발자국 스탬핑 + 깃발 플랜팅.
  **품질 게이트: 발주본(스플래시·만화) 옆에 놓고 위화감 없을 것 — 통과 전엔 지도에 안 올린다.**
  게이트가 늦으면 발자국(A)만 먼저 출시 가능(캐릭터 리스크 0).
- **눌리는 3D 스텝 노드**(듀오링고식): 노드에 아랫단(엣지) + 프레스 시 윗면 6px 침하.
  저빈도(레슨 진입)라 "고빈도 바운스 금지" 모션 규칙과 충돌 없음. 햅틱 탭(8)과 세트.
  **→ 2026-07-13 변형 탐색 중: 노드 실루엣을 발자국(장화 밑창)으로** — 아래 "진행 중 탐색" 섹션.
  확정되면 이 항목을 교체할 것(원형+상시 아랫단은 듀오링고 서명이라는 우려에서 출발).
- **대단원 탭 = 2글자 키워드 필**(로마숫자 보류 — 목차/시험범위 톤이라 부담, 사용자 피드백):
  과학 중1 = 탐구·생물·열·물질·힘·기체·우주. 전체 제목은 파란 히어로 카드가 소유(중복 해소).
- **앱바**: 그리드 아이콘 → "과학 ▾" 과목 칩(현재 과목 표시 + 과목 허브 진입 겸용).
  우상단 프로필 버튼은 마이 탭(goTab("my"))으로 통일(현재는 로그인 화면으로 가는 개편 전 잔재).

보류/별도 결정: C(징검다리 리스킨 — '밟는' 감각은 노드 버튼화가 절반 흡수), D(공책 두들 전면 리브랜딩).

## 진행 중 탐색 (2026-07-13) — 노드 실루엣을 발자국(장화 밑창)으로

사용자 우려("원형+상시 아랫단은 듀오링고 짭으로 보일 수 있다")에서 출발. 1차 반응 긍정("나쁘진
않네")이지만 **아직 확정 아님** — 다음 세션은 목업을 보여주고 확정받은 뒤 위 확정 묶음 3번을 갱신할 것.

- **목업 v2에 적용 완료**: 폰 데모 지도 노드 4개가 밑창 노드로 교체돼 있고(defs `#bsole` + use,
  프레스·걷기·상태 전환 전부 동작), "변형 탐색" trybox에 상태 견본 5종(열림·완료·잠금·좌우 기울임 쌍).
  원형 3D 버튼 박스는 "비교용 이전 안"으로 남김.
- **확정된 세부(사용자 지시)**: 발끝은 진행 방향(아래) — 위를 향하면 느낌표(!)로 읽힘. 맨발가락형
  금지, 장화 밑창(앞창+뒤꿈치 2조각) — 장화 레벨(core/level.ts BOOT_TIERS)과 세계관 직결.
- **간격·스크롤 실측**: 노드 중심 좌표를 원형과 동일하게 쓰면 지도 높이·스크롤 총량 불변(데모 중심
  150,58 / 85,170 / 195,270 / 110,372 — 원형과 동일). 노드가 세로 18px 길어 사이 여백만 줄어듦 —
  답답한지 여부만 눈 판정 항목.
- **스틱맨은 패드 위에 선다**: 발 위치 = 노드 중심 +8px(패드 중심), 걷기 경로 하향 보정
  (walkpath M85,178 → 195,278). 트레일 발자국 도장도 발끝이 진행 방향이라 노드와 한 문법.
- **구현 함정(목업에서 실제로 밟은 것)**: ① `.p-map svg` 전역(absolute·inset:0·100%) 규칙이 노드
  svg를 0×0으로 만든다 — `.bsn svg`에 position:static·width:auto 해제 필수(워커 svg와 같은 함정).
  ② 프레스 6px 침하 transform은 `.bs-top` 그룹 CSS에만 — use의 attribute transform과 섞으면 CSS가
  덮어씀(방향 회전은 `#bsole` 내부 rotate(180 24 35)가 소유). ③ 아이콘·금별 배지는 도형과 분리된
  그룹이라 발끝 방향과 무관하게 항상 정립.
- **남은 결정**: 지도 노드 좌·우 ±8° 기울임(리듬↑ vs 소란↑ — 견본은 trybox 마지막 쌍), 시험 노드
  잉크 밑창 유지 여부, 아티팩트 게시본 갱신(승인 대기 — 폰 확인용), 확정 시 이 README 3번 교체.
